While researching how to skip numbers in ordered lists, I found references saying "<ol start='5' />" has been deprecated as of HTML 4.01, and that we should us style sheets instead. So I looked that up, and found we can use ひらがな for list-style-type!!!

I'm making my default list style be hiragana from now on. Actually retroactive, cause I'm going to put the following STYLE block in my default css code:

     <STYLE type="text/css">
         OL             { list-style-type: hiragana }
	 OL.decimal     { list-style-type: decimal }
         OL.lower-roman { list-style-type: lower-roman }   
         OL.upper-roman { list-style-type: upper-roman }   
         OL.hiragana    { list-style-type: hiragana }
         OL.katakana    { list-style-type: katakana }   
     </STYLE>

With that, and the code below, we get the results below below. (*)

    <OL class="hiragana">
      <LI> This is the first item.
      <LI> This is the second item.
      <LI> This is the third item.
    </OL>

    <OL class="decimal">
      <LI> This is the first item.
      <LI> This is the second item.
      <LI> This is the third item.
    </OL>

    <OL class="katakana">
      <LI> This is the first item.
      <LI> This is the second item.
      <LI> This is the third item.
    </OL>

    <OL>
      <LI> This is item number 0000.
      <LI> This is item number 0001.
      <LI> This is item number 0002.
      ....
    </OL>

(*) Using hiragana as the default might not be the best idea in the world; when testing the other styles, georgian and armenian showed up as ? marks on my browser. So presumably it requires the characterset to be installed on someone's browser to work correctly. It *might* work in general because I've got a specification for the encoding type, but I'm not going to worry enough to test beyond what I've done already.
